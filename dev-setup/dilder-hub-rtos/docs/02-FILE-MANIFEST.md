# 02 — File Manifest: every file, explained

This document explains the **purpose of every file** in `dilder-hub-rtos/` — the ones we
wrote by hand, the ones carried over from the original firmware, the ones a tool
**generates**, and the ones the compiler **produces**. If you ever wonder "what is this
file and do I edit it?", it's here.

Files fall into four kinds. Each entry below is tagged:

- ✍️ **HAND-WRITTEN** — source we author and edit.
- 📦 **CARRIED-OVER** — proven code copied from the original `dilder-hub` (lightly or not
  yet modified); will be reorganized in Phase 2.
- ⚙️ **GENERATED** — produced by a tool at build time. Do **not** edit; it gets
  overwritten. (Usually git-ignored.)
- 🏗️ **BUILD OUTPUT** — produced by the compiler/linker into `build/`. Never edit; never
  commit.

## The overall directory structure

```
dilder-hub-rtos/
├── CMakeLists.txt                ✍️  build recipe (what to compile + link)
├── pico_sdk_import.cmake         📦  shim that locates the Pico SDK
├── FreeRTOS_Kernel_import.cmake  ✍️  shim that locates the FreeRTOS kernel
├── FreeRTOSConfig.h              ✍️  the RTOS settings (the heart of the RTOS build)
├── freertos_hooks.c              ✍️  kernel-required callbacks (memory + error hooks)
├── lwipopts.h                    ✍️  TCP/IP stack settings (tuned for the RTOS)
├── btstack_config.h              📦  Bluetooth stack settings
├── wifi_config.h                 ✍️  Wi-Fi SSID/pass placeholders + timezone
├── dilder.gatt                   📦  Bluetooth GATT database (source for a generated .h)
├── quotes.h                      ⚙️  the octopus's quotes (generated by the DevTool)
├── main.c                        ✍️  the firmware: state machine + UI task + drivers/sensors
├── rtos_tasks.c / rtos_tasks.h   ✍️  Phase 2: the Input/Display/Housekeeping task orchestration
├── bt.c / bt.h                   📦  Bluetooth LE peripheral logic
├── lib/                          📦  Waveshare display driver + fonts (self-contained copy)
│   ├── Config/                   – board pin wiring + low-level helpers
│   ├── e-Paper/                  – the e-ink panel drivers (V2/V3/V3a/V4)
│   ├── Fonts/                    – bitmap fonts
│   └── GUI/                      – Waveshare's paint helpers (mostly unused by us)
├── docs/                         ✍️  this documentation set
├── build/                        🏗️  USB build output (git-ignored)
└── build-ota/                    🏗️  Wi-Fi-OTA build output (git-ignored)
```

(After **Phase 2**, most of `main.c` moves into new `src/rtos/`, `src/drivers/`,
`src/app/`, and `src/net/` folders — each file small and single-purpose. This manifest
will be updated then.)

## Top-level files (the RTOS-specific ones come first)

### `CMakeLists.txt` ✍️
The build recipe. CMake reads it to learn which `.c` files to compile, which libraries to
link, and which compile-time options to set. The RTOS-specific lines: it `include()`s the
FreeRTOS import shim, links `FreeRTOS-Kernel` + `FreeRTOS-Kernel-Heap4` + `pico_flash`,
and swaps the Wi-Fi library from the *polled* variant to the *FreeRTOS* variant
(`pico_cyw43_arch_lwip_sys_freertos`). It also defines the firmware name
(`dilder_hub_rtos`) and the optional `PICOWOTA_OTA` over-the-air build. **Heavily
commented** — read it as a lesson in how an embedded build is wired.

### `FreeRTOSConfig.h` ✍️ — *the most important RTOS file*
A header full of `#define configXXX value` lines. FreeRTOS is configured entirely at
**compile time** (not at runtime), and this file is where we make every choice: **2
cores** (SMP), core affinity, the 1 kHz tick, the 128 KB memory pool (heap_4), priorities,
the ARMv8-M (Cortex-M33) options (FPU on, MPU/TrustZone off), and the Pico SDK interop
flags that make `sleep_ms()` cooperate with the scheduler. Every line is commented for a
newcomer. **If the RTOS misbehaves, look here first.**

### `freertos_hooks.c` ✍️
FreeRTOS calls back into a few functions with fixed names that **we** must provide, or the
program won't link. This file supplies them: the **error hooks** (out-of-memory,
stack-overflow → print and halt) and the **static-memory hooks** that hand the kernel RAM
for its internal idle/timer tasks (because we enabled static allocation). Pure boilerplate
the kernel demands — written as a teaching artifact about memory ownership.

### `FreeRTOS_Kernel_import.cmake` ✍️ (copied from the kernel, kept local)
A small CMake shim, mirroring `pico_sdk_import.cmake`. When `include()`d, it locates the
FreeRTOS kernel source (via `FREERTOS_KERNEL_PATH`), picks the correct CPU port for the
target chip (here `RP2350_ARM_NTZ` — the Cortex-M33 non-TrustZone port), and exposes the
`FreeRTOS-Kernel` / `FreeRTOS-Kernel-Heap4` CMake targets we link against.

### `pico_sdk_import.cmake` 📦
The equivalent shim for the Pico SDK: it reads `PICO_SDK_PATH` and pulls the SDK into the
build. Standard Raspberry Pi boilerplate, unchanged.

### `lwipopts.h` ✍️ (edited for the RTOS)
Settings for **lwIP**, the TCP/IP networking stack (DHCP, DNS, the NTP time client). The
key change from the original: `NO_SYS 0` — run lwIP **under the OS** in its own task,
instead of the bare-metal polled mode. Adds the tcpip-thread and mailbox sizes that OS
mode requires. Commented to explain the mode switch.

### `btstack_config.h` 📦
Configuration for **BTstack**, the Bluetooth Low Energy stack: enables BLE peripheral
mode, secure pairing, and sizes its buffers (kept small because the cyw43 chip shares one
SPI bus for Wi-Fi and Bluetooth). Unchanged from the original.

### `wifi_config.h` ✍️
Holds the default Wi-Fi `WIFI_SSID`/`WIFI_PASS` (kept as **placeholders** in git — real
credentials are passed via CMake `-D` flags, never committed) plus the NTP server and
timezone offset.

### `dilder.gatt` 📦
The **source** for the Bluetooth "GATT database" (the menu of services/characteristics a
phone sees). At build time `pico_btstack_make_gatt_header()` compiles this text file into
a generated C header (`dilder.h`, see below) that the firmware includes.

### `quotes.h` ⚙️ GENERATED
The octopus's hundreds of mood-tagged quote strings, as a C array. **Generated by the
DevTool** from the quote definitions in `tools/devtool/devtool.py`. Git-ignored and
regenerated; never hand-edit.

## The firmware logic

### `main.c` 📦 (the Phase 0/1 monolith)
Right now this single ~3900-line file **is** the firmware: hardware setup, the octopus
rendering, all the menu screens, Wi-Fi, the accelerometer, the battery gauge, and the main
state machine. The **only** RTOS change so far: instead of running its big loop directly,
`main()` creates **one** FreeRTOS task (`app_task`, pinned to core 0) that contains the
loop, and starts the scheduler. **Phase 2 splits this file apart** into the task/driver/app
modules; until then it stays whole so we can verify the RTOS plumbing in isolation.

### `rtos_tasks.c` / `rtos_tasks.h` ✍️ — *the Phase 2 task split*
The FreeRTOS orchestration, kept free of hardware/driver code (those stay in `main.c`). It
defines the four tasks and how they talk:
- **Input task** (core 0, top priority) — the only runtime joystick reader; debounces, does
  edge + auto-repeat, posts press *events* to `g_input_q`.
- **Display task** (core 1, alone) — the only post-boot e-ink caller; pops a finished buffer
  index off `g_render_q`, does the ~300 ms refresh, returns the buffer to `g_free_q`. Also
  registers core 1 with the flash lockout and signals the UI it's safe.
- **Housekeeping task** (core 0, low priority) — calls `hk_sample()` (in `main.c`) to read
  the accelerometer/steps/battery and publish a mutex-guarded `sensor_snapshot_t`.
- the **2-buffer free/render queue** hand-off (no shared-buffer race), the input queue, the
  snapshot mutex, the flash-ready semaphore, and `rtos_tasks_start()` which creates + pins
  all of them. The header is the contract; the UI in `main.c` calls `display_render()`,
  `ui_get_input()`, `rtos_snapshot_get()`.

### `bt.c` / `bt.h` 📦
The Bluetooth LE peripheral: advertises as "Dilder Hub", does passkey pairing, and exposes
a small custom service so a phone can read the current mood/steps and send a command. Talks
to BTstack.

## `lib/` — the Waveshare display library (self-contained copy)

The original firmware **symlinks** a shared `lib/`. This RTOS variant keeps its **own
copy** so it is fully self-contained *and* so we can modify the e-ink driver in Phase 3
(adding a hardware interrupt for the BUSY signal) without affecting other firmwares.

### `lib/Config/`
- `DEV_Config.c` / `DEV_Config.h` ✍️📦 — the **hardware abstraction**: maps the e-ink
  panel's pins (SPI, DC, RST, BUSY) to specific GPIOs and provides `DEV_Digital_Read`,
  `DEV_Delay_ms`, SPI transfer, etc. **This is where Phase 3's BUSY-interrupt hook lands**
  (it currently busy-waits via `DEV_Delay_ms(10)`).
- `Debug.h` 📦 — a `printf`-style debug macro toggle.
- `rtc_compat.h` 📦 — a tiny shim so the firmware's `datetime_t` clock API works the same
  on RP2040 (which has a hardware RTC) and RP2350 (which uses a software clock).

### `lib/e-Paper/`
The drivers for the 2.13" e-ink panel, one pair per hardware revision
(`EPD_2in13_V2/V3/V3a/V4`). We use **V4** (selected by `-DDISPLAY_VARIANT=V4`). Each
`.c/.h` knows the panel's command sequences and the **`ReadBusy()`** function that waits
~300 ms for a refresh — the exact bottleneck this whole RTOS effort is about.

### `lib/Fonts/`
Bitmap fonts (`font8`…`font24`) as C arrays of pixel data. The firmware actually uses its
own compact 5×7 font in `main.c`, so these are mostly legacy/unused but kept for
completeness and so the Waveshare `GUI_Paint` code compiles.

### `lib/GUI/`
`GUI_Paint.c/.h` — Waveshare's general-purpose drawing helpers (lines, shapes, strings).
The Dilder draws into its own framebuffer, so this is largely unused; carried over so the
library is intact.

### `lib/version.h` 📦
A firmware version string (`DILDER_VERSION`) shown on the device-info screen.

## Files you will see **generated** during a build (inside `build/`)

These appear after you run CMake/Ninja. All are 🏗️ **BUILD OUTPUT** or ⚙️ **GENERATED** —
never edit, never commit (they're git-ignored).

| File (in `build/`) | Kind | What it is |
|---|---|---|
| `dilder_hub_rtos.elf` | 🏗️ | The fully-linked program with debug symbols. The "real" binary; everything else is derived from it. |
| `dilder_hub_rtos.uf2` | 🏗️ | The flashable image. Drag-and-drop onto the BOOTSEL drive. |
| `dilder_hub_rtos.bin` / `.hex` | 🏗️ | Raw/hex variants of the image (alternate flashers). |
| `dilder_hub_rtos.dis` | 🏗️ | Disassembly (human-readable machine code) — for deep debugging. |
| `dilder_hub_rtos.elf.map` | 🏗️ | The linker "map": where every function/variable landed in memory and how big it is. Great for chasing size/RAM. |
| `generated/dilder_hub_rtos_gatt_header/dilder.h` | ⚙️ | The Bluetooth GATT database compiled from `dilder.gatt` into a C array (`profile_data`). `bt.c` includes it. |
| `pico-sdk/`, `FREERTOS_KERNEL/`, `picowota/` (subdirs) | 🏗️ | Per-build copies of the compiled SDK/kernel/bootloader objects. |
| `CMakeCache.txt`, `build.ninja`, `CMakeFiles/` | ⚙️ | CMake/Ninja's own bookkeeping. Delete the whole `build/` folder to start fresh. |

### In `build-ota/` (only when built with `-DPICOWOTA_OTA=ON`)

| File | Kind | What it is |
|---|---|---|
| `dilder_hub_rtos.elf` | 🏗️ | The **standalone app**, linked at flash offset `0x1005b000` (so the bootloader can place it). This is the **OTA payload** pushed over Wi-Fi. |
| `picowota_dilder_hub_rtos.elf` / `.bin` | 🏗️ | The **combined** bootloader + app image. |
| `picowota_dilder_hub_rtos.uf2` | 🏗️ | The combined image as a flashable `.uf2` — flash this **once** over USB to install the Wi-Fi updater + first app. |

## Files **outside** this folder that the build depends on

| Path | Why it matters |
|---|---|
| `../../FreeRTOS-Kernel/` | The RTOS kernel source (a git submodule, with its own nested port submodule). |
| `../../picowota/` | The Wi-Fi OTA bootloader (a git submodule), compiled into the combined image. |
| `$PICO_SDK_PATH` (e.g. `~/pico/pico-sdk`) | The Pico SDK — GPIO, Wi-Fi, USB, BTstack, lwIP. |
| `tools/devtool/devtool.py` | Generates `quotes.h` and provides the build/flash/OTA GUI; now lists `dilder-hub-rtos`. |

## Quick rule of thumb

- **Edit:** `CMakeLists.txt`, `FreeRTOSConfig.h`, `freertos_hooks.c`, `lwipopts.h`,
  `wifi_config.h`, `main.c`, `bt.*`, `lib/Config/DEV_Config.*`, the `docs/`.
- **Never edit (regenerated/built):** `quotes.h`, anything in `build/` or `build-ota/`,
  the generated `dilder.h`.
- **Rarely touch (carried-over libraries):** the rest of `lib/`, `btstack_config.h`,
  `pico_sdk_import.cmake`.

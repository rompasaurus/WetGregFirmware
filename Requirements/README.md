# WetGreg — Requirements

Feature requirement documents and implementation plans. Each file follows the
naming convention:

```
Feature_<Name>_Status-<Incomplete|Complete>.md
```

A feature is marked **Complete** (and renamed) once it is implemented, verified,
and merged. Each document contains: an agile user story, acceptance criteria, an
implementation plan grounded in the codebase, a status / definition-of-done
section, and a suggested git branch name.
